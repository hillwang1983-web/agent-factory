# AI-native 5G 核心网 Agent 工厂成本模型修正版

评估日期：2026-06-07

目的：修正此前偏传统研发组织的成本口径，按 AI Agent 原生开发模式重新评估 5G 核心网功能开发工厂的成本。

## 一、核心修正

之前的评估把成本拆成：

```text
人类 HITL + worker + CI + Agent 平台维护 + token + 实验室
```

这个拆法仍然像传统研发组织，只是把“程序员”替换成了“Agent”。AI-native 模式应该改成：

```text
成本 = 智能循环成本 + 真实验证成本 + 人类裁决成本 + 外部物理资源成本
```

其中：

| 成本项 | 本质 | 主要计量单位 |
| --- | --- | --- |
| 智能循环成本 | Agent 理解、规划、写代码、改测试、调试、review、整理证据。 | token / Agent loop |
| 真实验证成本 | 编译、测试、仿真、压测、长稳、抓包、制品归档。 | validation run |
| 人类裁决成本 | 只处理 contract、L2/L3、发布承诺、安全边界。 | decision event |
| 外部物理资源成本 | RAN/UE/IMS/达梦/加密卡/国密 USIM/SM4 对端。 | availability window |

这意味着：worker、CI、平台维护不应该先被看作固定成本，而应该被看作 **Agent loop 的执行后端**。如果已有机器，它们前期可以不折算现金成本；只有当验证队列成为瓶颈时，才转化为扩容成本。

## 二、AI-native 成本单位

### 2.1 ADL：Agent Delivery Loop

一个 ADL 是 Agent 完成一次可验证推进的闭环：

```text
读取上下文 -> 生成或修改测试 -> 修改代码 -> 执行验证 -> 分析失败 -> 产出证据
```

不同需求消耗的不是“人天”，而是 ADL 数量。

| 需求类型 | 典型 ADL | 说明 |
| --- | ---: | --- |
| 配置/OAM/指标 | 3-8 ADL | 多数失败可由 Agent 自修。 |
| 普通后端/API/NMS | 6-15 ADL | 涉及 contract、前后端和测试。 |
| 核心网控制面 | 12-30 ADL | AMF/SMF/UDM/AUSF/PCF，协议和状态复杂。 |
| UPF/5G-LAN/HA/数据面 | 25-80 ADL | 数据面、pcap、性能、长稳、故障注入。 |
| 安全/国产化/外部硬件 | 30-100+ ADL | SDK、真实设备、供应商、不可控失败多。 |

### 2.2 VU：Validation Unit

一个 VU 是一次真实验证动作：

| VU 类型 | 例子 | 是否应先折算现金 |
| --- | --- | --- |
| local build | `ninja -C build` | 有闲置机器时不必。 |
| unit test | CUnit/pytest/TS test | 有闲置机器时不必。 |
| integration | Open5GS + MongoDB + UERANSIM | 记录占用，不先算钱。 |
| chaos/long-run | HA、DB failover、UPF 压测 | 若占用长时间机器，需要算容量。 |
| L3 external | RAN/UE/IMS/达梦/加密卡 | 必须单独算资源窗口。 |

### 2.3 HDE：Human Decision Event

人类不按“人月”算，而按必须裁决的事件算。

| HDE 类型 | 例子 | 典型耗时 |
| --- | --- | ---: |
| contract freeze | 指标、告警、API、配置字段确认。 | 0.5-2 h |
| high-risk review | UPF/HA/安全/数据库/互操作关键变更。 | 1-4 h |
| L2/L3 decision | 外部资源不到位时是否按 L2 交付。 | 0.5-1.5 h |
| release gate | 是否发布、限制说明是否可接受。 | 1-3 h |
| incident decision | P0/P1 自愈超过阈值后的根因裁决。 | 1-4 h |

## 三、从零建设成本的新口径

### 3.1 MVP 工厂

目标：让 AI Agent 可以独立完成 2-5 个 5GC 需求的 L2 软件侧闭环。

| 项 | AI-native 口径 |
| --- | --- |
| 建设周期 | 3-7 天，不是 2-4 周。 |
| 主要成本 | token + 1-2 人裁决事件 + 已有机器验证。 |
| 现金预算 | 1万-5万 RMB，如果已有机器和工具链。 |
| 适用 | 验证方法、跑通 ADL/VU/HDE 流程。 |

MVP 不应该先建设复杂平台。文件型 registry、worktree、prompt 模板、证据目录、脚本化验证就够。

### 3.2 可用工厂

目标：每周稳定交付若干 5GC 软件侧需求，形成可复用 Context Pack、Contract 和 evidence。

| 项 | AI-native 口径 |
| --- | --- |
| 建设周期 | 2-4 周。 |
| 主要成本 | token 消耗、Context Pack 沉淀、验证队列、少量 HDE。 |
| 现金预算 | 10万-40万 RMB，如果不新建实验室。 |
| 适用 | 中低风险需求常态交付。 |

这个阶段才需要 Review Agent、Evidence Agent、成本看板、验证队列和自动 L2/L3 分类。

### 3.3 生产工厂

目标：可以持续发布 5GC 功能，支持复杂核心网需求和一定规模并发。

| 项 | AI-native 口径 |
| --- | --- |
| 建设周期 | 4-8 周。 |
| 主要成本 | 大量 ADL、验证资源调度、Context/Contract 资产化、HDE 队列。 |
| 现金预算 | 30万-100万 RMB，不含新购大型实验室。 |
| 适用 | 多需求并行、发布包、回归、客户可验收软件侧交付。 |

生产工厂的价值不是“平台很贵”，而是让后续每个需求的 ADL 下降、HDE 下降、返工下降。

### 3.4 L3 实验室工厂

目标：真实 RAN/UE/IMS/达梦/加密卡/国密/SM4 等外部资源可长期联调。

| 项 | AI-native 口径 |
| --- | --- |
| 建设周期 | 取决于物理资源，不是 AI 决定。 |
| 主要成本 | 外部资源 availability window。 |
| 现金预算 | 100万-500万+ RMB，视已有实验室而定。 |
| 适用 | 真实客户验收、国产化、安全硬件、复杂互操作。 |

这部分不是 AI 工厂本体成本，而是 5GC 真实世界验证成本。

## 四、月度运行成本的新口径

### 4.1 如果已有机器和基础 CI

| 成本项 | 月度成本 |
| --- | ---: |
| token / Agent loop | 0.5万-10万 RMB |
| 机器执行 | 现金可按 0 处理，只记录占用率。 |
| 存储 | 0.05万-0.5万 RMB |
| 人类裁决 | 1万-8万 RMB，按 HDE 发生量。 |
| 外部资源 | 0-按需。 |

这种情况下，token 很可能就是主要新增现金成本。

### 4.2 如果需要云上弹性验证

| 成本项 | 月度成本 |
| --- | ---: |
| token / Agent loop | 1万-20万 RMB |
| 云 worker / 容器 | 0.5万-5万 RMB |
| 存储和制品 | 0.1万-1万 RMB |
| 人类裁决 | 2万-12万 RMB |
| 外部资源 | 0-20万+ RMB |

### 4.3 如果是企业多团队

| 成本项 | 月度成本 |
| --- | ---: |
| token / Agent loop | 5万-50万+ RMB |
| 验证后端 | 5万-30万 RMB |
| 存储和审计 | 1万-10万 RMB |
| 人类裁决 | 10万-50万 RMB |
| 实验室资源 | 20万-100万+ RMB |

## 五、为什么仍然要建工厂

如果 token 才是主要成本，那更需要工厂，而不是更不需要工厂。

因为工厂的目标是降低每个需求的 ADL：

| 没有工厂 | 有工厂 |
| --- | --- |
| Agent 反复读全仓库。 | Context Pack 复用。 |
| 每个需求重新定义字段。 | Contract 复用。 |
| 失败靠人看日志。 | Debugger Agent + 失败分类。 |
| PR 都找人审。 | Review Agent 先筛。 |
| 验收证据每次手工整理。 | Evidence harness 自动生成。 |
| token 消耗不可预测。 | ADL/VU/HDE 可计量。 |

所以 AI 工厂的 ROI 是：

```text
用一次性 Context/Contract/测试资产建设
换取长期每个需求更少 ADL、更少 HDE、更少返工、更高发布可信度
```

## 六、最终修正版结论

跳出传统研发模式后，建议这样评估：

| 阶段 | 建设周期 | 现金成本，已有机器 | 现金成本，需云资源 | 核心成本 |
| --- | ---: | ---: | ---: | --- |
| MVP | 3-7 天 | 1万-5万 | 3万-10万 | token + 少量 HDE |
| 可用工厂 | 2-4 周 | 10万-30万 | 15万-40万 | ADL 资产化 |
| 生产工厂 | 4-8 周 | 30万-80万 | 40万-100万 | ADL/VU/HDE 管理 |
| L3 实验室 | 1-6 个月 | 视已有资源 | 100万-500万+ | 外部物理资源 |

一句话：

> AI-native 成本评估不应先问“需要几个人、几台机器、几个月”，而应先问“一个需求需要多少 ADL、多少 VU、多少 HDE，其中多少可以复用和自动化”。  
> 如果已有机器，前期主要新增成本确实应该是 token，而不是 worker 或平台维护。
