# AI Agent 工厂建设成本评估

评估日期：2026-06-07

关联文档：

- `docs/ai-agent-development-environment-from-zero.md`
- `docs/core-network-2.11-token-usage-estimate.md`
- `docs/core-network-2.11-8human-hh-estimate.md`

评估目标：估算从零搭建一套可支撑核心网产品 2.11 开发交付的 AI Agent 工厂所需成本，并区分“工厂建设成本”和“2.11 冲刺运行成本”。

## 一、结论

### 1.1 成本总览

| 口径 | 适用场景 | 一次性建设成本 | 2.11 冲刺运行成本 | 合计参考 |
| --- | --- | ---: | ---: | ---: |
| 最小可行版 | 当前 Mac + 2 台 Linux worker，先跑 2-5 个需求试点。 | 5万-15万 RMB | 8万-20万 RMB | 13万-35万 RMB |
| 推荐生产版 | 4-8 台 worker，完整 Context/Contract/CI/自愈/证据体系，支撑 45-55 Agent 槽位。 | 12万-35万 RMB | 18万-55万 RMB | 30万-90万 RMB |
| 全量 L3 实验室版 | 推荐生产版 + 真实 RAN/UE/IMS/达梦/加密卡/国密/SM4 联调环境。 | 30万-120万+ RMB | 35万-100万+ RMB | 65万-220万+ RMB |

推荐按 **生产版 + L2 保底 + 关键 L3 补证** 建设：

> 一次性建设预算：**12万-35万 RMB**  
> 2.11 冲刺运行预算：**18万-55万 RMB**  
> 总预算建议：**30万-90万 RMB**，另设外部硬件/License 风险池。

如果外部实验室资源已经具备，成本会明显下降；如果达梦、加密卡、国密 USIM、SM4 基站侧、IMS/4G/5G 真实环境都需要新采购或租赁，成本主要会上升在外部资源，不是模型 token。

## 二、成本结构

| 成本项 | 性质 | 是否必须 | 说明 |
| --- | --- | --- | --- |
| 模型 API / Agent token | 运行成本 | 必须 | 用于需求拆解、代码生成、测试、debug、review、证据。 |
| Agent 编排平台 | 建设成本 | 必须 | 包括 prompts、ADU registry、Context Pack、调度和状态管理。 |
| Linux worker / CI 算力 | 建设 + 运行 | 必须 | Open5GS 构建、仿真、集成、Chaos、长稳。 |
| 制品和日志存储 | 运行成本 | 必须 | build log、test log、pcap、evidence、release bundle。 |
| 人类 HITL | 运行成本 | 必须 | 8 人只做架构、安全、环境、发布和验收门禁。 |
| 外部联调资源 | 建设 + 运行 | L3 必须 | 达梦、加密卡、国密 USIM、IMS、真实 RAN/UE、SM4 对端。 |
| 安全与合规 | 建设成本 | 必须 | secret 隔离、脱敏、审计、权限控制。 |
| 运维监控 | 运行成本 | 建议 | Agent run 监控、预算阈值、失败率、队列拥塞。 |

## 三、一次性建设成本

### 3.1 最小可行版

目标：2-3 天内建起可试跑环境，验证 Agent 工厂方法是否可用。

| 项目 | 内容 | 成本估算 |
| --- | --- | ---: |
| 工厂目录和 registry | `.ai-agent/`、requirements、ADU、policy、evidence 模板。 | 0.5万-1.5万 |
| Prompt 和 Agent 模板 | Context、Contract、TestWriter、Developer、Build、Debug、Evidence。 | 1万-3万 |
| 基础 CI | build、unit、integration mock、evidence check。 | 1万-3万 |
| 2 台 worker | 可用现有机器则为 0；云资源/临时服务器按 3-4 周计。 | 0.5万-2万 |
| 初始 Context Pack | `common`、`oam-observability`、`5glan-ha`。 | 1万-3万 |
| 安全边界 | secret 规则、可写目录、日志脱敏、权限策略。 | 0.5万-1.5万 |
| 试点验证 | KPI + 5G-LAN 两个 ADU 走通闭环。 | 1万-2万 |
| 合计 | 最小工厂可运行。 | 5万-15万 |

### 3.2 推荐生产版

目标：支撑 2.11 全量需求的 40-70 Agent 槽位并发。

| 项目 | 内容 | 成本估算 |
| --- | --- | ---: |
| ADU 调度体系 | 队列、状态机、冲突域、worktree 策略、失败重试策略。 | 2万-5万 |
| 全量 Context Pack | 7-8 个 lane 上下文包，含代码地图和风险清单。 | 2万-5万 |
| 全量 Contract | metrics、alarms、OAM API、License、5G-LAN、HA、DB、Crypto。 | 2万-6万 |
| 自愈闭环 | TestWriter/Build/TestRunner/Debugger/Chaos/Evidence 自动循环。 | 3万-8万 |
| CI/CD 和 evidence harness | PR 模板、日志归档、验收矩阵、known issues、release bundle。 | 2万-5万 |
| 4-8 台 worker | build/unit/integration/chaos worker，3-4 周云资源或本地折旧。 | 1万-5万 |
| 监控和成本控制 | token budget、Agent 成功率、失败队列、缓存命中率。 | 0.5万-2万 |
| 安全和审计 | secret 扫描、权限隔离、脱敏、审计日志。 | 1万-4万 |
| 合计 | 可支撑 2.11 全量 Agent 工厂。 | 12万-35万 |

### 3.3 全量 L3 实验室版

目标：不仅软件侧完成，还尽量完成真实环境验收。

| 项目 | 成本估算 | 说明 |
| --- | ---: | --- |
| 真实 5G RAN/UE/测试卡 | 5万-30万+ | 如果已有实验室资源，可降为 0。 |
| 4G/EPC/IMS 环境 | 3万-20万+ | 可用 SIPp/mock 降级为 L2。 |
| 达梦数据库和 HA 环境 | 3万-20万+ | License、服务器、适配支持需单独确认。 |
| 江南信安加密卡 | 2万-15万+ | 取决于型号、SDK、厂商支持。 |
| 国密 USIM/卡商支持 | 1万-10万+ | 包括测试卡、参数、互通支持。 |
| SM4 N3 对端/基站侧配合 | 5万-30万+ | 非标准改造风险高。 |
| 实验室维护与联调人力 | 5万-20万+ | 环境搭建、排障、补证。 |
| 合计 | 30万-120万+ | 供应商报价差异会很大。 |

## 四、模型 token 成本

### 4.1 价格口径

按 OpenAI 官方 API 价格页在 2026-06-07 可见价格：

| 模型 | 输入 | 缓存输入 | 输出 |
| --- | ---: | ---: | ---: |
| GPT-5.5 | $5.00 / 1M token | $0.50 / 1M token | $30.00 / 1M token |
| GPT-5.4 | $2.50 / 1M token | $0.25 / 1M token | $15.00 / 1M token |
| GPT-5.4 mini | $0.75 / 1M token | $0.075 / 1M token | $4.50 / 1M token |

说明：OpenAI 官方价格页同时说明 Batch API 可节省 50% 输入和输出成本，API 与 ChatGPT 订阅分开计费。

### 4.2 2.11 运行 token 假设

沿用已有估算：

| 交付口径 | 总 token | 推荐用途 |
| --- | ---: | --- |
| L2 软件侧全量 | 50M-70M | 全部需求软件侧完成，硬件依赖 mock。 |
| 推荐交付 | 70M-100M | L2 全量 + 关键主线 L3 + 部分硬件补证。 |
| 冲刺全量 L3 | 100M-160M | 真实环境齐备，接受更多联调/debug 循环。 |

推荐将 70M-100M 拆为：

| 类型 | 70M 场景 | 100M 场景 |
| --- | ---: | ---: |
| 未缓存输入 | 35M | 50M |
| 缓存输入 | 25M | 35M |
| 输出 | 10M | 15M |

### 4.3 模型费用估算

| 模型策略 | 70M token | 100M token | 160M token |
| --- | ---: | ---: | ---: |
| 全部 GPT-5.4 mini | $73 | $107 | $174 |
| 全部 GPT-5.4 | $244 | $359 | $590 |
| 全部 GPT-5.5 | $400 | $718 | $1,178 |
| 推荐混合：20% GPT-5.5 + 55% GPT-5.4 + 25% mini | $249 | $366 | $602 |

折合人民币按 1 USD = 7.2 RMB 粗算：

| 模型策略 | 70M token | 100M token | 160M token |
| --- | ---: | ---: | ---: |
| 全部 GPT-5.4 mini | 约 525 RMB | 约 770 RMB | 约 1,253 RMB |
| 全部 GPT-5.4 | 约 1,755 RMB | 约 2,584 RMB | 约 4,248 RMB |
| 全部 GPT-5.5 | 约 2,880 RMB | 约 5,166 RMB | 约 8,482 RMB |
| 推荐混合 | 约 1,793 RMB | 约 2,635 RMB | 约 4,334 RMB |

结论：模型 token 不是最大成本。真正的大头是人类 HITL、worker/环境、外部实验室资源和供应商支持。

### 4.4 建设期 token

仅搭建 Agent 工厂，不跑完整 2.11，实现最小到生产版通常消耗：

| 建设阶段 | Token |
| --- | ---: |
| 最小可行版 | 3M-8M |
| 推荐生产版 | 8M-20M |
| 加上 2 个试点 ADU | 12M-30M |

按推荐混合模型，建设期模型费约为 **300-1,200 RMB**。这部分很小，不能成为建设瓶颈。

## 五、算力和基础设施成本

### 5.1 Linux worker 成本

| 档位 | 配置 | 3-4 周成本估算 |
| --- | --- | ---: |
| 最小 | 2 台 8 vCPU / 16-32 GB | 0.5万-2万 RMB |
| 推荐 | 4-8 台 8-16 vCPU / 16-64 GB | 1万-5万 RMB |
| 高压 | 8-12 台 16-32 vCPU / 64 GB+ | 4万-12万 RMB |

说明：

- 如果使用现有本地服务器，现金成本可接近 0，但要计入排队、维护和故障成本。
- 如果使用云主机，核心变量是是否 24 小时运行、是否按需实例、是否需要公网带宽和大盘。
- 如果长稳、压力、Chaos 测试较多，worker 成本会明显高于普通 CI。

### 5.2 存储和制品

| 项目 | 成本估算 |
| --- | ---: |
| 日志、pcap、core dump、build artifacts | 500-3,000 RMB / 月 |
| 制品仓库或对象存储 | 300-2,000 RMB / 月 |
| 备份和快照 | 300-2,000 RMB / 月 |

建议按 1-3 TB 可用空间准备。pcap、core dump、长稳日志要设置生命周期，避免成本失控。

### 5.3 Agent 编排和监控

| 项目 | 成本估算 |
| --- | ---: |
| 自建脚本 + 本地看板 | 0.5万-2万 RMB 一次性 |
| 使用现成平台/商业 Agent IDE | 需按席位或企业报价确认 |
| 监控告警 | 0.2万-1万 RMB / 月 |

如果已有 Codex、Antigravity、Claude Code、Gemini CLI、OpenClaw 等工具，可以优先用现有工具组合，不必先购买完整商业平台。

## 六、人类 HITL 成本

### 6.1 2.11 冲刺期 HH

已有评估推荐：

| 交付口径 | 人类 HH |
| --- | ---: |
| L2 软件侧全量 | 280-380 HH |
| 推荐交付 | 360-520 HH |
| 尽量全量 L3 | 520-760 HH |

### 6.2 按小时成本折算

| 人员综合成本 | 360 HH | 520 HH | 760 HH |
| --- | ---: | ---: | ---: |
| 300 RMB/h | 10.8万 | 15.6万 | 22.8万 |
| 500 RMB/h | 18.0万 | 26.0万 | 38.0万 |
| 800 RMB/h | 28.8万 | 41.6万 | 60.8万 |

推荐用 **500 RMB/h** 做预算中线：

> L2 全量 + 关键 L3：**18万-26万 RMB 人类 HITL 成本**。

如果 8 人是内部已有团队，也可以把这部分看成机会成本；但它仍然是真实瓶颈，因为架构、安全和验收门禁不能完全外包给 Agent。

## 七、2.11 冲刺运行成本

### 7.1 推荐交付口径

| 成本项 | 估算 |
| --- | ---: |
| 模型 token | 0.2万-0.6万 RMB |
| Linux worker / CI / 存储 | 1.5万-6万 RMB |
| Agent 运行监控和编排维护 | 1万-4万 RMB |
| 人类 HITL | 18万-26万 RMB |
| 外部资源补证 | 0万-20万 RMB |
| 合计 | 20.7万-56.6万 RMB |

这里的推荐口径假设：L2 全量完成，关键主线争取 L3，外部资源已有一部分或以短期租用/厂商支持方式补证。

### 7.2 全量 L3 口径

| 成本项 | 估算 |
| --- | ---: |
| 模型 token | 0.4万-1万 RMB |
| Linux worker / CI / 存储 | 4万-12万 RMB |
| Agent 编排和维护 | 2万-8万 RMB |
| 人类 HITL | 26万-61万 RMB |
| 外部实验室/硬件/License | 30万-120万+ RMB |
| 合计 | 62万-202万+ RMB |

全量 L3 的成本不由 AI 决定，主要由真实硬件、专有数据库、加密卡、基站侧配合和供应商响应速度决定。

## 八、成本控制策略

| 策略 | 可节省项 | 预期效果 |
| --- | --- | --- |
| Context Pack 复用 | token | 节省 20%-35% token。 |
| Contract 先冻结 | token + HH | 减少返工和跨 lane 冲突。 |
| Agent 文件范围白名单 | token + 合并成本 | 减少误读全仓和误改文件。 |
| 先 L2 mock 后 L3 补证 | 外部环境 + HH | 避免硬件早期拖慢开发。 |
| 失败 3 轮进 Human Gate | token + AH | 防止 Agent 无限自愈。 |
| worker 自动关停 | 云资源 | 非长稳任务不 24 小时空跑。 |
| pcap/core 生命周期 | 存储 | 避免日志无限增长。 |
| mini 模型处理低风险任务 | token | 文档、证据、简单 review 可降本。 |
| Batch API 处理非即时任务 | token | 异步 review、文档摘要、证据整理可降本。 |

## 九、预算建议

### 9.1 建设预算

建议先批 **15万-25万 RMB** 建设预算：

| 用途 | 金额 |
| --- | ---: |
| `.ai-agent` 工厂骨架、ADU、Context、Contract | 4万-8万 |
| CI/worker/evidence harness | 4万-8万 |
| 自愈闭环和试点 ADU | 3万-6万 |
| 安全、监控、成本控制 | 2万-3万 |
| 缓冲 | 2万-4万 |

### 9.2 2.11 冲刺预算

建议另批 **25万-60万 RMB** 运行预算：

| 用途 | 金额 |
| --- | ---: |
| 人类 HITL | 18万-26万 |
| worker/CI/存储 | 2万-8万 |
| Agent 运行维护 | 2万-6万 |
| 模型 token | 0.3万-1万 |
| 外部资源补证 | 0万-20万 |
| 缓冲 | 3万-10万 |

### 9.3 外部资源风险池

如果目标是更多 L3 真实验收，建议单独设 **30万-100万+ RMB** 风险池，不与 Agent 工厂预算混用。

风险池用途：

| 用途 | 说明 |
| --- | --- |
| 达梦数据库/HA 环境 | License、支持、服务器、迁移验证。 |
| 加密卡 | 板卡、SDK、厂商支持。 |
| 国密 USIM | 卡、参数、卡商互通支持。 |
| SM4 N3 | 基站侧或对端改造、性能验证。 |
| 真实 4G/5G/IMS 环境 | RAN/UE/IMS/测试仪表/现场支持。 |

## 十、最终判断

建立这套 AI Agent 工厂本身并不贵，真正需要控制的是外部环境和人类门禁负载。

推荐决策：

1. 先按 **15万-25万 RMB** 建设生产可用的 Agent 工厂。
2. 2.11 冲刺按 **25万-60万 RMB** 准备运行预算。
3. 模型 token 预算按 **100M token** 控制，现金成本只需约数千 RMB，但必须做预算阈值和缓存策略。
4. 外部硬件/License 单独设风险池，不能让它吞掉 Agent 工厂预算。
5. 如果 6 月 8 日前外部资源不到位，相关需求自动按 L2 软件侧完成，L3 作为补证项。
