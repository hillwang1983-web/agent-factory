目标口径：

- AI Agent 是主执行体；8 人人类团队只承担 HITL：架构裁决、契约评审、安全审计、外部资源协调、关键 PR 审核、验收确认。
- 使用 HH（Human Hours）估算人类投入；使用 AH（Agent Hours）估算 Agent 云端运行时间。

## 一、结论

### 1.1 推荐估算

| 交付口径 | Agent AH | 人类 HH | 代码量 | Token |
| --- | ---: | ---: | ---: | ---: |
| L2 软件侧全量完成 | 1,050-1,450 AH | 280-380 HH | 78k-123k LOC | 50M-70M |
| L2 全量 + 关键主线 L3 | 1,350-1,950 AH | 360-520 HH | 95k-140k LOC | 70M-100M |
| 尽量全量 L3 | 1,850-2,800 AH | 520-760 HH | 130k-168k+ LOC | 100M-160M |

推荐按 **L2 全量 + 关键主线 L3** 执行：

> **Agent：1,350-1,950 AH**  
> **人类：360-520 HH**  
> 8 人平均：**45-65 HH/人**，约 3 周内每人每天 2-4 小时高密度 HITL。

### 1.2 与 Antigravity 评估的关系

Antigravity 文档给出：

- 28 个需求；
- 849 AH；
- 214 HH；
- 约 91k LOC；
- 8 人人类团队。

本评估校准为：

- 34 条全量需求；
- 代码控制线 95k-140k LOC；
- 增加 open5gs-nms/OAM、mock/provider、证据包、部署脚本、全量需求矩阵；
- 因此推荐值上调到 **1,350-1,950 AH / 360-520 HH**。

差异不是因为 AI 变慢，而是因为本口径覆盖更多需求、更多证据、更多外部依赖和更完整的发布包。

## 三、Agent 2.0 自愈架构下的 AH/HH 模型

采用测试一体化自愈双环：

1. **TestWriter Agent**：先生成 CUnit、接口测试、gNBSim/UERANSIM/SIPp、mock 硬件用例。
2. **Developer Agent**：按 contract 和测试红灯实现代码。
3. **BuildAgent**：持续编译，自愈头文件、符号、构建依赖、类型错误。
4. **TestRunner Agent**：跑单测、接口、回归、仿真、容器环境。
5. **Debugger Agent**：用 GDB/Valgrind/日志诊断 crash、内存、竞态和超时。
6. **ChaosAgent**：执行主备切换、链路故障、压力、长稳和恢复验证。
7. **Evidence Agent**：生成验收证据包和限制说明。

| Agent 类型 | AH 占比 | 推荐 AH |
| --- | ---: | ---: |
| TestWriter | 12%-16% | 180-280 |
| Developer | 36%-42% | 520-760 |
| BuildAgent | 8%-12% | 120-220 |
| TestRunner | 14%-18% | 210-340 |
| Debugger | 14%-18% | 210-350 |
| ChaosAgent | 5%-8% | 75-150 |
| Evidence/Release Agent | 5%-8% | 75-150 |
| 合计 | 100% | 1,350-1,950 |

## 七、人类介入门禁

AI Agent 自愈可以处理编译错误、单测失败、普通 crash、常规重构；以下情况必须进入 HH 预算：

| 触发条件 | 介入角色 | 单次 HH |
| --- | --- | ---: |
| contract 变更影响 2 个以上 lane | H1 + 相关专家 | 1-3 |
| P0 crash 自愈超过 3 轮 | H1/H2/H3/H7 | 1-4 |
| 多线程/HA/UPF 数据面出现不可解释性能退化 | H3/H7 | 2-6 |
| 加密卡/达梦/国密/SM4 真实环境失败 | H5/H6/H7 | 2-8 |
| 需求只能 L2，无法 L3 | H1/H8 + 领域专家 | 1-3 |
| 发布门禁不通过 | H1/H8/H7 | 2-6 |

## 八、最终建议

1. 采用 Antigravity 2.0 的 **TestWriter + BuildAgent + TestRunner + Debugger + ChaosAgent** 自愈模式。
2. 8 人人类团队不要参与普通编码和普通 debug，只做 HITL 审查和物理世界约束处理。
3. 推荐预算设为 **1,350-1,950 AH + 360-520 HH**。
4. 若 6 月 8 日前外部资源不到位，硬件/现场依赖项自动从 L3 改为 L2，避免 HH 被无底洞式联调吞掉。
5. 每个 PR 必须携带测试日志、GDB/Valgrind 或等价诊断、evidence bundle；否则人类不审。
