# 通用 5G 核心网 AI Agent 开发工厂成本评估

评估日期：2026-06-07

评估口径：跳出 2.11 版本和 2026-06-30 发布约束，评估从零开始建立一套可长期支撑 5G 核心网需求开发、测试、验收和发布的 AI Agent 开发工厂所需成本。

关联文档：

- `docs/ai-agent-development-environment-from-zero.md`
- `docs/ai-agent-factory-cost-estimate.md`
- `docs/ai-agent-factory-minimal-hitl-evaluation.md`

## 一、结论

如果不绑定 2.11 和 6 月 30 日，成本模型会从“冲刺型项目成本”变成“平台型建设成本”。

推荐按三档建设：

| 档位 | 目标 | 建设周期 | 一次性建设成本 | 月度运行成本 | 最小 HITL |
| --- | --- | ---: | ---: | ---: | ---: |
| MVP 试点版 | 可完成低/中风险 5GC 需求的软件侧 L2 交付。 | 2-4 周 | 15万-40万 RMB | 3万-10万 RMB/月 | 2-3 人 |
| 生产可用版 | 可稳定完成多数 5GC 需求开发、测试、证据和发布。 | 6-10 周 | 60万-150万 RMB | 12万-35万 RMB/月 | 3-4 人 |
| 企业/实验室版 | 可支撑多版本并行、真实 5GC/4G/IMS/安全国产化 L3 验收。 | 3-6 个月 | 200万-600万+ RMB | 40万-150万+ RMB/月 | 5-8 人 |

推荐路线：

> 先建设 **生产可用版**，预算按 **60万-150万 RMB 一次性建设**、**12万-35万 RMB/月运行** 准备。  
> 如果已有 5GC 实验室、RAN/UE、IMS、达梦、加密卡、国密 USIM 等资源，可明显降低企业版成本。  
> 如果这些资源都要从零购买和集成，企业/实验室版通常会进入 **200万-600万+ RMB** 区间。

这里的成本大头不是模型 token，而是：

1. 5GC 专业 Context Pack、Contract、测试资产和自愈流水线建设。
2. Linux worker、仿真环境、CI、制品和证据系统。
3. 5GC/RAN/UE/IMS/数据库/安全硬件实验室。
4. 少量高水平 HITL owner。
5. 长期维护、回归、版本演进和合规审计。

## 二、能力边界

### 2.1 这个工厂应该能做什么

| 能力 | 说明 |
| --- | --- |
| 需求分析 | 把 5GC 需求拆成 ADU、接口契约、验收断言和风险等级。 |
| 代码开发 | 支持 AMF、SMF、UPF、UDM、AUSF、PCF、NMS/OAM、测试工具和脚本开发。 |
| 测试先行 | 自动生成单测、接口测试、仿真测试、回归测试和 mock。 |
| 构建自愈 | 自动处理编译、链接、依赖、类型和常规构建问题。 |
| 调试自愈 | 自动分析日志、core dump、GDB/Valgrind、pcap 和失败测试。 |
| 仿真验收 | 使用 UERANSIM、gNBSim、SIPp、mock DB、mock HSM 完成 L2 验收。 |
| 真实验收 | 对接真实 RAN/UE/IMS/数据库/安全硬件完成 L3 补证。 |
| 发布 | 自动生成 evidence bundle、验收矩阵、发布说明、回滚方案和已知问题。 |

### 2.2 不应该默认承诺什么

| 不默认承诺 | 原因 |
| --- | --- |
| 所有需求都自动 L3 | L3 依赖真实网络、硬件、License 和供应商配合。 |
| 完全无人值守发布 | 架构、安全、客户承诺和 L2/L3 判定必须有人类门禁。 |
| 任何 5GC 私有协议都开箱即用 | 需要逐步积累 Context Pack、Contract 和测试向量。 |
| 立即替代完整研发组织 | 更现实的定位是高吞吐交付工厂 + 少量专家门禁。 |

## 三、建设成本拆分

### 3.1 MVP 试点版

目标：证明 AI Agent 工厂能端到端完成 5GC 需求开发，不追求全域覆盖。

| 模块 | 内容 | 成本 |
| --- | --- | ---: |
| Agent 工厂骨架 | ADU registry、prompt、worktree、run logs、evidence 模板。 | 3万-8万 |
| 基础 Context Pack | Open5GS 代码地图、构建、测试、OAM/NMS、UPF/SMF 基础上下文。 | 3万-8万 |
| 最小 Contract | metrics、alarm、OAM API、配置 schema、测试断言。 | 2万-5万 |
| 最小 CI/worker | 2 台 worker、build、unit、mock integration。 | 2万-6万 |
| 自愈闭环 MVP | TestWriter、Developer、BuildAgent、TestRunner、Evidence。 | 3万-8万 |
| 安全边界 | secret 隔离、日志脱敏、Agent 可写路径。 | 1万-3万 |
| 试点需求 | 2-5 个代表需求端到端完成。 | 1万-2万 |
| 合计 | 可验证方法论。 | 15万-40万 |

适合：

- 建设初期；
- 内部验证；
- L2 软件侧交付；
- 非高风险 OAM、指标、配置、轻量核心网功能。

### 3.2 生产可用版

目标：成为团队日常 5GC 功能开发和发布的主通道。

| 模块 | 内容 | 成本 |
| --- | --- | ---: |
| Agent 编排平台 | ADU 队列、状态机、冲突域、并发槽位、失败升级、人类队列。 | 8万-20万 |
| 全域 Context Pack | AMF/SMF/UPF/UDM/AUSF/PCF/OAM/NMS/安全/DB/HA/测试。 | 8万-20万 |
| Contract 体系 | API、配置、指标、告警、License、QoS、HA、DB、Crypto、证据。 | 8万-20万 |
| 自愈开发体系 | TestWriter、Developer、BuildAgent、TestRunner、Debugger、Chaos、Evidence。 | 10万-25万 |
| CI/CD 和发布 | PR 门禁、回归矩阵、evidence harness、release bundle、回滚方案。 | 8万-18万 |
| 仿真和 L2 环境 | UERANSIM、gNBSim、SIPp、MongoDB、mock DB、mock HSM、mock RAN/UE。 | 6万-15万 |
| Worker 和存储 | 4-8 台 worker、日志、pcap、core dump、制品仓库。 | 5万-15万 |
| 安全和审计 | secret 管理、脱敏、权限、审计、合规检查。 | 4万-12万 |
| 运营看板 | token、Agent 成功率、失败率、HITL 队列、成本、吞吐。 | 3万-8万 |
| 合计 | 日常生产可用。 | 60万-150万 |

适合：

- 长期持续开发；
- 每月多个需求并行；
- 多个 Agent lane；
- L2 全量交付 + 关键路径 L3 补证；
- 少量 HITL 管理。

### 3.3 企业/实验室版

目标：支持多版本、多产品线、真实环境验收、客户交付和合规审计。

| 模块 | 内容 | 成本 |
| --- | --- | ---: |
| 多租户 Agent 平台 | 多项目、多分支、多权限、多模型、多环境调度。 | 30万-80万 |
| 5GC 专业知识库 | 3GPP、内部设计、历史缺陷、测试手册、客户场景、协议证据库。 | 20万-60万 |
| 高级测试平台 | 自动拓扑、场景编排、流量回放、长稳、压测、Chaos、pcap 分析。 | 30万-100万 |
| 真实 5GC 实验室 | RAN/UE、SIM、4G/5G/IMS、UPF 性能、专线或射频环境。 | 50万-200万+ |
| 国产化/安全实验室 | 达梦、加密卡、国密 USIM、mTLS/国密 TLS、SM4 对端。 | 30万-150万+ |
| 企业 CI/CD | 制品签名、SBOM、漏洞扫描、审计、灰度、版本追踪。 | 20万-60万 |
| Agent 治理 | prompt 版本、模型路由、评测集、红队、安全策略、预算控制。 | 20万-50万 |
| 运维和支持体系 | 7x12/7x24 支持、失败恢复、容量管理、供应商管理。 | 20万-60万 |
| 合计 | 企业级长期能力。 | 200万-600万+ |

适合：

- 多版本持续交付；
- 客户现场前真实验收；
- 高可靠、高安全和国产化要求；
- 多团队共用。

## 五、模型 token 成本

按 2026-06-07 可见 OpenAI 官方 API pricing 口径，模型成本可按输入、缓存输入、输出三类估算。Agent 工厂长期运行时，Context Pack 和共享 contract 会提高缓存输入比例，因此 token 成本通常低于直觉。

### 5.1 月度 token 用量

| 运行规模 | 月度 token | 说明 |
| --- | ---: | --- |
| 试点 | 10M-30M | 2-5 个需求、低并发、少量 debug。 |
| 生产常态 | 50M-200M | 每月多个需求、回归、证据和发布。 |
| 高强度发布月 | 200M-800M | 多版本并行、复杂联调、真实环境补证。 |
| 企业多团队 | 800M-2B+ | 多产品线、多项目、多实验室长期运行。 |

### 5.2 粗略费用

按混合模型策略估算：

| 运行规模 | 模型费用 |
| --- | ---: |
| 10M-30M token/月 | 100-1,000 RMB/月 |
| 50M-200M token/月 | 1,000-8,000 RMB/月 |
| 200M-800M token/月 | 4,000-40,000 RMB/月 |
| 800M-2B+ token/月 | 2万-10万+ RMB/月 |

模型费仍然不是最大头。只要 Context Pack、缓存、文件范围和 Review Agent 设计得好，成本中心主要在测试环境和专家门禁。

## 六、单需求边际成本

长期工厂建成后，开发一个 5GC 需求的边际成本会下降。

| 需求类型 | 典型例子 | Agent token | Agent AH | 人类 HH | 边际成本 |
| --- | --- | ---: | ---: | ---: | ---: |
| 简单配置/OAM | 页面字段、指标、告警、配置校验 | 1M-5M | 10-40 | 2-8 | 0.3万-2万 |
| 中等核心网功能 | QoS、License、地址池、NMS + 后端 | 5M-20M | 40-120 | 8-24 | 2万-8万 |
| 复杂协议/数据面 | 5G-LAN、UPF、PFCP、HA、互操作 | 20M-80M | 120-400 | 24-80 | 8万-30万 |
| 安全/国产化/外部硬件 | 加密卡、国密、达梦、SM4、IMS/4G/5G L3 | 30M-120M | 200-700 | 40-160 | 15万-80万+ |

边际成本主要由需求是否触碰以下因素决定：

| 因素 | 成本影响 |
| --- | --- |
| 核心数据面 | 需要性能、稳定性、pcap 和长稳。 |
| 状态同步/HA | 需要故障注入和一致性证据。 |
| 外部硬件 | 需要供应商、SDK、真实环境和排障。 |
| 安全密钥 | 需要合规、人类审查和脱敏。 |
| 客户验收 | 需要发布包、回放证据和限制说明。 |

## 七、HITL 配置

长期工厂不需要固定 8 人，但需要稳定 owner。

| 规模 | 人员配置 | 说明 |
| --- | --- | --- |
| MVP | 2-3 人 | 工厂 owner + 核心网专家 + QA/环境可合并。 |
| 生产常态 | 3-4 人 | 工厂/架构、核心网协议、平台安全、QA发布。 |
| 企业/实验室 | 5-8 人 | 增加安全国产化、实验室、客户验收、平台运维。 |

生产常态推荐 4 人：

| 角色 | 主要职责 |
| --- | --- |
| P1 Agent 工厂负责人 / 总架构 | ADU、contract、scope、发布口径。 |
| P2 核心网协议与数据面专家 | AMF/SMF/UPF/PFCP/5G-LAN/HA/互操作。 |
| P3 平台、安全与数据专家 | OAM/NMS、DB、License、TLS、国密、加密卡。 |
| P4 QA / 环境 / 发布证据专家 | CI、仿真、Chaos、回归、evidence、L2/L3。 |

## 八、建设路线

### Phase 0：准备和盘点

周期：1-2 周

成本：5万-15万 RMB

输出：

- 代码、测试、文档、环境 inventory；
- 需求类型分类；
- Agent 安全边界；
- 初始 ADU 模型；
- MVP 试点范围。

### Phase 1：MVP 工厂

周期：2-4 周

成本：15万-40万 RMB

输出：

- `.ai-agent` 工厂骨架；
- 2-3 个 Context Pack；
- 最小 Contract；
- build/test/evidence 流水线；
- 2-5 个试点需求闭环。

### Phase 2：生产可用

周期：6-10 周

成本：60万-150万 RMB

输出：

- 全域 Context Pack；
- 全域 Contract；
- 自愈闭环；
- Review Agent；
- evidence harness；
- L2 仿真环境；
- 3-4 人 HITL 队列；
- 可发布版本流程。

### Phase 3：实验室和企业治理

周期：3-6 个月

成本：200万-600万+ RMB

输出：

- 真实 L3 实验室；
- 多版本并行；
- 安全和国产化测试；
- 供应商资源接入；
- 企业审计和平台运维；
- 长期评测集和质量基线。

## 九、成本控制建议

| 建议 | 作用 |
| --- | --- |
| 不先买大而全平台 | 先用 Codex/Agent runtime + 自建 registry/evidence 验证方法。 |
| 先 L2 后 L3 | 软件闭环先跑通，真实硬件作为补证。 |
| Context Pack 产品化 | 这是长期降 token、降误判、降返工的核心资产。 |
| Contract 作为第一等产物 | 前后端、核心网、测试、证据都围绕 contract。 |
| Review Agent 前置 | 让人类只看高风险，不看所有普通 PR。 |
| 真实实验室分阶段建设 | 先 UERANSIM/gNBSim/SIPp，再 RAN/UE/IMS/安全硬件。 |
| 以需求类型定价 | 简单 OAM 与 UPF/HA/安全硬件不能用同一成本模型。 |
| 建立长期评测集 | 每次模型、prompt、工具升级都用评测集回归。 |

## 十、最终建议

如果目标是建立一套长期可复用、可以持续完成 5G 核心网需求开发并发布功能的 AI Agent 开发工厂，建议预算这样设：

| 阶段 | 预算 |
| --- | ---: |
| Phase 0 盘点 | 5万-15万 RMB |
| Phase 1 MVP | 15万-40万 RMB |
| Phase 2 生产可用 | 60万-150万 RMB |
| Phase 3 企业/实验室 | 200万-600万+ RMB |

最现实路线：

1. 先投入 **20万-50万 RMB** 做 Phase 0 + Phase 1，证明 2-5 个真实需求可以端到端交付。
2. 试点成功后投入 **60万-150万 RMB** 建成生产可用版。
3. 每月按 **12万-35万 RMB** 运营生产版工厂。
4. 只有在明确需要大量 L3 真实验收时，再投入 **200万-600万+ RMB** 做企业/实验室版。

一句话判断：

> **一个能真正发布 5G 核心网功能的 AI Agent 工厂，生产可用版不是几万元级别的小工具，而是约 60万-150万 RMB 的平台建设；但建成后，单个普通需求的边际交付成本会显著下降。**
